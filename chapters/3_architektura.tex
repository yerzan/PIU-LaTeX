\chapter{Implementacja HTML, CSS i JavaScript}

\section{Zakres wdrożenia i responsywność}
Zgodnie z wymogami projektu, wybrane widoki zaprojektowane w programie Figma zostały zaimplementowane do działającego kodu przy użyciu technologii HTML5 oraz CSS3. 

Kluczowym założeniem podczas kodowania było zapewnienie pełnej responsywności (RWD - Responsive Web Design). Dzięki zastosowaniu nowoczesnych technik ostylowania (takich jak Flexbox oraz CSS Grid) a także zapytań medialnych (Media Queries), zaimplementowane interfejsy płynnie skalują się i dostosowują swój układ do szerokości okna przeglądarki. Potwierdza to, że system jest w pełni gotowy do obsługi zarówno na urządzeniach desktopowych, jak i mobilnych (smartfony, tablety).

Zgodnie z wymogiem prowadzącego, w procesie tworzenia bazowej struktury HTML oraz podstawowych reguł CSS wykorzystano wsparcie narzędzi AI (np. ChatGPT), przy czym wygenerowany kod został ręcznie zoptymalizowany, przetestowany i dostosowany do rygorystycznych wymogów responsywności.

\section{Struktura katalogów i plików}
Aby zachować maksymalną czytelność i porządek w kodzie źródłowym, projekt został podzielony na logiczne sekcje odpowiadające wdrożonym widokom. Poniżej przedstawiono zaktualizowaną strukturę plików zaimplementowanej części interfejsu:

\begin{itemize}
    \item \textbf{MainPage} -- Strona główna aplikacji.
    \begin{itemize}
        \item \texttt{freshcurated.html} -- semantyczna struktura widoku strony startowej i oferty.
    \end{itemize}
    
    \item \textbf{Dashboard} -- Panel użytkownika.
    \begin{itemize}
        \item \texttt{dashboard.html} -- struktura widoku z historią zamówień i aktywnymi dietami.
    \end{itemize}

    \item \textbf{OrderConfigurator} -- Zaawansowany konfigurator zamówienia.
    \begin{itemize}
        \item \texttt{index.html} -- struktura formularzy, wyboru celów i kalendarza dostaw.
        \item \texttt{style.css} -- główny arkusz stylów oparty na zmiennych (CSS Variables) dla utrzymania spójności palety kolorów, stylujący siatki (Grid/Flexbox) oraz elementy UI.
        \item \texttt{script.js} -- plik z logiką interakcji i obsługą formularza zamówienia.
    \end{itemize}

    \item \textbf{AdminDiets} -- Moduł zarządzania dla administratora.
    \begin{itemize}
        \item \texttt{Diets-menu-managment.html} -- struktura widoku panelu administracyjnego do zarządzania biblioteką posiłków.
    \end{itemize}

    \item \textbf{ClientProfile} -- Moduł CRM.
    \begin{itemize}
        \item \texttt{client-profile.html} -- kompleksowy widok profilu klienta (CRM) agregujący dane o zamówieniach, restrykcjach dietetycznych (alergie) oraz historię aktywności.
    \end{itemize}
\end{itemize}

Takie podejście ułatwia ewentualną dalszą rozbudowę aplikacji oraz integrację front-endu z systemami back-endowymi.
